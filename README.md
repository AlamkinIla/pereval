Проект реализует REST API для приёма и обработки информации о горных перевалах.
API предназначено для использования мобильным приложением туристов и
внутренними сервисами Федерации спортивного туризма России (ФСТР).

---

Турист с помощью мобильного приложения отправляет данные о новом перевале.
Данные сохраняются в базе данных PostgreSQL и проходят модерацию сотрудниками ФСТР.

Для каждого перевала хранится статус модерации:
- `new` — запись добавлена
- `pending` — запись взята в работу модератором
- `accepted` — модерация прошла успешно
- `rejected` — данные отклонены
